Defining full-text search properties

Tracker provides nonstandard full-text search capabilities, in order to use these, the string properties can use tracker:fulltextIndexed: 

ex:name a rdf:Property;
        rdfs:domain ex:Mammal;
        rdfs:range xsd:string;
        tracker:fulltextIndexed true;
        tracker:weight 10.

Weighting can also be applied, so certain properties rank higher than others in full-text search queries. With tracker:fulltextIndexed in place, sparql queries may use full-text search capabilities: 

SELECT { ?mammal a ex:Mammal;
                 fts:match "timmy" }
